# Cache OmniRoute model catalog for interactive startup

OmniRoute model discovery can be slow or unavailable at Pi startup, but built-in Pi providers avoid this by shipping a generated model catalog with the release. We decided that interactive Pi sessions should register OmniRoute from the last valid local Model Catalog Cache immediately, then run a best-effort Discovery Refresh in the background and replace the cache/provider only when the live result is valid. Metadata commands such as `pi --list-models` do not read this cache; this trades possibly stale model selection during TUI startup for a fast and resilient interactive experience.
